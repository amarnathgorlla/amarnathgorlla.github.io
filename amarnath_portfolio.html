03 — Engineering Decisions

Problems I hit while building solo, and how I closed them.

Building alone means every blocker is yours to diagnose. A few of the real ones from shipping Swarama:

BRANDING
Problem: Product had no name or identity going into build.
Solved: Named it "Swarama" — Telugu-inspired for "connecting" / "calling for great work" — and designed the logo, icons, and a cream + dark-orange app theme in Canva.
BUILD ENV
Problem: Expo Go builds kept failing on-device with package-version mismatches and dependency-version imbalances.
Solved: Traced the conflicts to mismatched Expo SDK / native module versions and pinned compatible versions to unblock local builds.
TESTING
Problem: Intermittent network connectivity failures while testing on-device through Expo Go.
Solved: Diagnosed and resolved the connectivity issue to get reliable on-device test sessions.
COST
Problem: Google Maps Platform required a billing account and paid API usage to keep mapping features live.
Solved: Replaced it with an OpenStreetMap-based mapping module — kept full location and routing functionality at zero recurring cost.
